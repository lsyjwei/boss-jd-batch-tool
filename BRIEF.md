# BRIEF — BOSS直聘职位信息拉取小工具

> 一页内说清「做什么、为什么、怎么算成功」。立项后填完 success criteria 才能标记为 `active`。

## 背景

需要将 BOSS 直聘上的 JD（职位描述）和具体岗位页面信息提供给 Cursor 使用（辅助写简历、分析岗位等）。目前通过对话逐条上传链接或截图，效率太低，需要批量导入能力。

## 目标

一个小工具，支持**批量**导入 BOSS 直聘岗位信息（链接和/或截图），输出 Cursor 可直接读取的结构化文本或文件（如 Markdown / JSON），便于在 Agent 对话中 `@` 引用。

## 约束

- 输入形式：岗位链接列表、页面截图（批量）
- 输出形式：结构化 JD 文件，放在项目内固定目录，供 Cursor 读取
- 时间：优先 MVP（能批量处理 5–10 条即可验证）
- 不做：需要登录态爬取、做 Web UI、是否对接 BOSS API（如有可对接）
- 合规：注意 BOSS 直聘 ToS；优先「用户已打开的页面 / 用户提供的截图与 HTML」而非未授权爬取

## Success Criteria

- [x] 支持一次提交多个岗位链接或批量截图，无需逐条对话上传
- [x] 每条岗位输出包含：标题、公司、薪资（若有）、JD 正文、来源链接（若有）
- [x] 输出文件可被 Cursor `@` 引用，且 Agent 能读懂岗位要点
- [x] 提供 README：如何使用、输入输出示例
